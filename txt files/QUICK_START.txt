================================================================================
STEP 1: FIREBASE DATABASE RULES (One-time, 5 minutes)
================================================================================

1. Go to: https://console.firebase.google.com/
2. Click on project: "shazam-coffee"
3. Go to: Build ‚Üí Realtime Database
4. Click the "Rules" tab at the top
5. Delete ALL existing code
6. Copy-paste this code:

{
  "rules": {
    "orders": {
      ".read": true,
      ".write": true,
      "$orderId": {
        ".validate": "newData.hasChildren(['id', 'itemName', 'customerName', 'customerPhone', 'orderTime', 'endTime', 'status'])"
      }
    }
  }
}

7. Click the BLUE "Publish" button
8. Wait 1-2 seconds for confirmation
9. DONE! ‚úÖ

================================================================================
TROUBLESHOOTING
================================================================================

Q: Orders not appearing on other devices?
A: Check Firebase rules are PUBLISHED (see Step 1)
   Make sure you clicked the blue PUBLISH button

Q: Getting "Permission denied" error?
A: Rules aren't published yet
   Go back to Step 1 and check rules are saved

Q: Orders not updating in real-time?
A: Refresh the page manually
   Check internet connection
   Clear browser cache (Ctrl+Shift+Delete)

Q: Can't login to barista dashboard?
A: Check credentials (they're case-sensitive!)
   Clear browser cache
   Try Incognito/Private mode

Q: Still seeing old localStorage orders?
A: This is normal - Firebase will take over
   Clear browser data
   Or open in Incognito mode
   Orders will sync automatically
